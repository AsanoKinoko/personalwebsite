Constraints

Constraints là gì?

Constraints là cách phần tử "bám" vào Frame cha khi resize

Hiểu đơn giản: Khi bạn kéo to / nhỏ frame (mobile → tablet → desktop), element sẽ di chuyển / co giãn theo.

Constraints trong Figma rất quan trọng và được sử dụng thường xuyên trong việc thiết kế responsive khi thay đổi kích cỡ giao diện. Constraints hay dùng trong trường hợp UI đơn giản, không cần responsive phức tạp (ví dụ các app trên mobile).

2 chiều Constraints

Figma chia thành 2 trục:

  • Horizontal (ngang)
    • Left
    • Right
    • Left + Right
    • Center
    • Scale
  • Vertical (dọc)
    • Top
    • Bottom
    • Top + Bottom
    • Center
    • Scale

Giải thích từng loại (QUAN TRỌNG):

  • LEFT: Element luôn giữ khoảng cách với bên trái

    Dùng khi:

    • Icon menu
    • Logo
    • Avatar trái
  • RIGHT: Element giữ khoảng cách với bên phải

    Dùng khi:

    • Nút settings
    • Icon close
    • Cart icon
  • LEFT + RIGHT: Element giãn theo chiều ngang

    Nghĩa là: Frame to → element to theo. Frame nhỏ → element nhỏ theo

    Dùng:

    • Input field
    • Button full width
    • Card stretch
  • SCALE: Element sẽ scale theo % của frame

    Ví dụ: Frame gấp đôi → element gấp đôi, không giữ margin cố định nữa

    Dùng:

    • Background image
    • Responsive illustration

    CẢNH BÁO: Dev KHÔNG thích kiểu này lắm (khó control)

  • Với Vertical cũng khá tương tự:
    • Top: dính trên
    • Bottom: dính dưới
    • Top + Bottom: stretch dọc
    • Center: giữa
    • Scale: scale theo chiều dọc

Một số ví dụ thực tế (dễ hiểu):

  • Case 1: Button ở giữa màn hình

    Constraints:

    • Horizontal: Center
    • Vertical: Center
  • Case 2: Header bar
    • Logo: Left + Top
    • Avatar: Right + Top
    • Title: Center + Top
  • Case 3: Input full width
    • Horizontal: Left + Right
    • Vertical: Top
  • Case 4: Footer stick dưới
    • Vertical: Bottom

Một số sai lầm phổ biến:

  • Dùng Scale bừa bãi: Làm layout bị vỡ khi resize
  • Không set constraints: Resize → UI bay lung tung
  • Dùng Center cho mọi thứ: Không thực tế (UI thật hiếm khi center hết)

Lưu ý:

Constraints KHÔNG phải responsive hoàn chỉnh

  • Muốn chuẩn xịn → phải dùng:
    • Auto Layout
    • Hug / Fill container
  • Constraints chỉ là: "fallback system" cho layout cũ
