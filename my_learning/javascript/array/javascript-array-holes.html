"Holes" (phần tử trống) và undefined

Holes là chỉ số thiếu property: VD new Array(3) tạo [ <3 empty items> ].

  • arr[0] ở đây trả undefined, nhưng '0' in arr cho false (vì không có property).
  • undefined là giá trị có thật được gán: '0' in arr sẽ là true.

Hệ quả với method:

  • forEach, map, filter, every, some, find, reduce bỏ qua holes (không gọi callback cho index thiếu).
  • map/filter giữ cấu trúc (độ dài không đổi với map).
  • join() coi hole như "" (nhưng vẫn chèn dấu phân cách), còn toString() về bản chất là join(',').
const a = Array(3);             // [ <3 empty> ]

a.map(() => 1);                  // [ <3 empty> ] — callback không chạy

Array.from(a, () => 1);          // [1,1,1] — lấp lỗ

'0' in a;                        // false
