1. Cấu trúc kiểu trong Object Manipulation

Các utility types phổ biến:

Cấu trúc Ý nghĩa
? Thuộc tính tùy chọn
readonly Không thể gán lại
Record<K, V> Tạo kiểu map động
Partial<T> Làm cho tất cả thuộc tính trong T thành tùy chọn
Required<T> Ngược lại - tất cả bắt buộc
Pick<T, K> Lấy ra vài thuộc tính trong T
Omit<T, K> Bỏ đi vài thuộc tính trong T
ReturnType<F> Lấy kiểu trả về của hàm
Parameters<F> Lấy kiểu tham số của hàm

VD sử dụng utility types:

type User = { id: string; name: string; age: number };

type UserPreview = Pick<User, 'id' | 'name'>;
// { id: string; name: string}

type UserUpdate = Partial<User>;
// { id?: string; name?: string; age?: number }

type UserWithoutAge = Omit<User, 'age'>;
// { id: string; name: string}
