3. "as const" - Đóng băng literal types

Bình thường:

const cfg = { mode: 'dark', retry: 3 };
cfg.mode = 'light'; // hợp lệ
// TypeScript suy luận: type of cfg: { mode: string; retry: number }

Khi thêm as const:

const cfg = { mode: 'dark', retry: 3 } as const;
// Suy luận kiểu sẽ là: type of cfg: { readonly mode: 'dark'; readonly retry: 3 }

Nghĩa là:

  • Thuộc tính readonly → không thể gán lại.
  • Giá trị 'dark' và 3 được giữ nguyên là literal types, không bị nới thành string hay number.

VD khác:

const COLORS = ['red', 'green', 'blue'] as const;
// type: readonly ['red', 'green', 'blue']

type Color = typeof COLORS[number];
// => 'red' | 'green' | 'blue'

// => Rất hữu ích khi định nghĩa enum-like hoặc cấu hình cố định.
