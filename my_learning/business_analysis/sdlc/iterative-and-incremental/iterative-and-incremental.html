Iterative and Incremental Model (Mô hình lặp gia tăng)

Là mô hình kết hợp giữa:

  • Incremental: xây dựng hệ thống theo từng phần (feature/module), mỗi phần là một chức năng hoàn chỉnh nhỏ
  • Iterative: Xây hệ thống bằng cách lặp lại nhiều vòng (iteration), mỗi vòng cải tiến dần hệ thống

Hiểu đơn giản: Mỗi lần bạn thêm tính năng mới (Increment) sẽ đồng thời cải tiến tính năng cũ (Iteration)

Quy trình

Mỗi vòng (iteration/sprint):

  1. Chọn một số feature (increment)
  2. Phân tích + thiết kế
  3. Code
  4. Test
  5. Nhận feedback
  6. Improve (iteration)

Cứ thế lặp lại liên tục.

Đặc điểm quan trọng

  • Phát triển từng phần (Incremental): Mỗi lần build thêm feature mới
  • Cải tiến liên tục (Iterative): Refactor, optimize, UX improve
  • Feedback sớm & liên tục: User test từng version
  • Không cần hoàn hảo từ đầu: Build nhanh ➔ sửa dần

Ưu điểm

  • Ra sản phẩm sớm: Có thể release ngay từ Sprint 1
  • Linh hoạt cực cao: Dễ dàng đổi yêu cầu
  • Feedback liên tục: User tham gia từ đầu
  • Giảm risk: Sai sửa sớm
  • Improve liên tục: Code + UX tốt dần

Nhược điểm

  • Khó quản lý nếu team yếu: Dễ loạn
  • Scope creep: Khách hàng cứ thêm feature
  • Không có plan rõ từ đầu: Khó estimate
  • Technical debt: Nếu không refactor

Ví dụ một số rủi ro và cách xử lý:

  • Risk: Mở rộng phạm vi dự án (Scope creep)
    Fix: Ưu tiên các công việc còn tồn đọng (Prioritize backlog)
  • Risk: Mã nguồn lộn xộn (Code messy)
    Fix: Refactor mỗi iteration
  • Risk: Không có kiến trúc
    Fix: Thiết kế high-level từ đầu

Khi nào nên dùng?

  • GẦN NHƯ LUÔN LUÔN dùng khi:
    • Web app (React, NextJS, Spring Boot)
    • Mobile app
    • Startup
    • Product evolving
  • KHÔNG nên dùng khi:
    • Requirement cố định 100%
    • Project government / military
