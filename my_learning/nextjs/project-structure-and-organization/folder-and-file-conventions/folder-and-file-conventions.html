Quy ước về thư mục và tệp tin (Folder and file conventions)

1. Thư mục cấp cao nhất (Top-level folders)

Các thư mục cấp cao nhất được sử dụng để tổ chức mã nguồn và các tài nguyên tĩnh của ứng dụng.

  • app: App Router
  • pages: Pages Router
  • public: Tài nguyên tĩnh phục vụ. Ví dụ như ảnh, icons svg...
  • src: Thư mục nguồn ứng dụng tùy chọn

2. Các file cấp cao nhất (Top-level files)

Các file cấp cao nhất được sử dụng để:

  • Cấu hình ứng dụng
  • Quản lý dependencies
  • Chạy proxy
  • Tích hợp các công cụ giám sát
  • Định nghĩa các biến môi trường
File Mô tả
next.config.js File cấu hình cho Next.js
package.json Dependencies của dự án và các script
instrumentation.ts File OpenTelemetry và Instrumentation
proxy.ts Proxy request của Next.js
.env Biến môi trường (không nên được theo dõi bởi hệ thống quản lý phiên bản)
.env.local Biến môi trường cục bộ (không nên được theo dõi bởi hệ thống quản lý phiên bản)
.env.production Biến môi trường cho production (không nên được theo dõi bởi hệ thống quản lý phiên bản)
.env.development Biến môi trường cho development (không nên được theo dõi bởi hệ thống quản lý phiên bản)
eslint.config.mjs File cấu hình cho ESLint
.gitignore Các file và thư mục Git sẽ bỏ qua
next-env.d.ts File khai báo TypeScript cho Next.js (không nên chỉnh sửa trực tiếp)
tsconfig.json File cấu hình cho TypeScript
jsconfig.json File cấu hình cho JavaScript

3. Các file định tuyến (Routing Files)

Thêm page để tạo một route công khai, layout để tạo giao diện dùng chung như header, navigation, hoặc footer, loading để hiển thị skeleton loading, error để xử lý lỗi (error boundaries), và route cho API.

layout.tsx

  • Bắt buộc phải có
  • Là layout gốc
  • Phải chứa <html> và <body>

page.tsx

  • Là nội dung của route
  • Mỗi thư mục route phải có page.tsx
File Phần mở rộng Mô tả
layout .js .jsx .tsx Layout
page .js .jsx .tsx Trang
loading .js .jsx .tsx Giao diện khi đang tải
not-found .js .jsx .tsx Giao diện khi không tìm thấy
error .js .jsx .tsx Giao diện lỗi
global-error .js .jsx .tsx Giao diện lỗi toàn cục
route .ts Endpoint API
template .js .jsx .tsx Layout được render lại
default .js .jsx .tsx Trang fallback cho parallel route

4. Nested routes (Route lồng nhau)

Các thư mục xác định các segment của URL. Khi lồng thư mục thì các segment cũng được lồng theo. Layouts ở bất kỳ cấp nào sẽ bao bọc (wrap) các segment con của nó. Một route sẽ trở thành public khi có file page hoặc route.

Path URL pattern Ghi chú
app/layout.tsx — Root layout bao bọc tất cả các route
app/blog/layout.tsx — Bao bọc /blog và các route con
app/page.tsx / Route công khai
app/blog/page.tsx /blog Route công khai
app/blog/authors/page.tsx /blog/authors Route công khai

5. Dynamic routes (Route động)

Có thể tạo các segment động bằng dấu ngoặc vuông.

  • [segment] → một tham số
  • [...segment] → bắt tất cả (catch-all)
  • [[...segment]] → bắt tất cả nhưng tùy chọn

Có thể truy cập giá trị thông qua prop params.

Path URL pattern
app/blog/[slug]/page.tsx /blog/my-first-post
app/shop/[...slug]/page.tsx /shop/clothing, /shop/clothing/shirts
app/docs/[[...slug]]/page.tsx /docs, /docs/layouts-and-pages, /docs/api-reference/use-router

6. Route groups và thư mục private

Có thể tổ chức code mà không thay đổi URL bằng cách dùng route groups (group) và đặt các tệp không thể định tuyến vào các thư mục riêng tư (_folder).

Path URL pattern Ghi chú
app/(marketing)/page.tsx / Nhóm bị bỏ qua trong URL
app/(shop)/cart/page.tsx /cart Chia sẻ layout trong (shop)
app/blog/_components/Post.tsx — Không thể truy cập qua route; nơi an toàn cho UI utilities
app/blog/_lib/data.ts — Không thể truy cập qua route; nơi an toàn cho utilities
