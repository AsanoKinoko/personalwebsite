Installation

Tạo một ứng dụng Next.js mới và chạy nó trên máy cục bộ.

Bắt đầu nhanh (Quick start)

  1. Tạo một ứng dụng Next.js mới có tên my-app
  2. cd my-app và khởi động dev server.
  3. Truy cập: http://localhost:3000

pnpm

pnpm create next-app@latest my-app --yes
cd my-app
pnpm dev

npm

npx create-next-app@latest my-app --yes
cd my-app
npm run dev

yarn

yarn create next-app@latest my-app --yes
cd my-app
yarn dev

bun

bun create next-app@latest my-app --yes
cd my-app
bun dev
  • --yes sẽ bỏ qua các câu hỏi (prompt) bằng cách sử dụng các tùy chọn đã lưu hoặc giá trị mặc định. Thiết lập mặc định sẽ bật TypeScript, Tailwind, ESLint, App Router và Turbopack, với import alias là @/*.

Yêu cầu hệ thống (System requirements)

Trước khi bắt đầu, hãy đảm bảo môi trường phát triển của bạn đáp ứng các yêu cầu sau:

  • Phiên bản Node.js tối thiểu: 20.9
  • Hệ điều hành: macOS, Windows (bao gồm WSL) và Linux

Trình duyệt được hỗ trợ

Next.js hỗ trợ các trình duyệt hiện đại mà không cần cấu hình thêm.

  • Chrome 111+
  • Edge 111+
  • Firefox 111+
  • Safari 16.4+

Tìm hiểu thêm về hỗ trợ trình duyệt, bao gồm cách cấu hình polyfills và nhắm mục tiêu các trình duyệt cụ thể: https://nextjs.org/docs/architecture/supported-browsers

Tạo bằng CLI

Cách nhanh nhất để tạo một ứng dụng Next.js mới là sử dụng create-next-app, công cụ này sẽ tự động thiết lập mọi thứ cho bạn.

Để tạo một dự án, chạy:

  • pnpm: pnpm create next-app
  • npm: npx create-next-app@latest
  • yarn: yarn create next-app
  • bun: bun create next-app

Sau khi cài đặt, bạn sẽ thấy các câu hỏi sau:

What is your project named? my-app
Would you like to use the recommended Next.js defaults?
    Yes, use recommended defaults - TypeScript, ESLint, Tailwind CSS, App Router, Turbopack
    No, reuse previous settings
    No, customize settings - Choose your own preferences

Nếu bạn chọn tùy chỉnh cài đặt, bạn sẽ thấy các câu hỏi sau:

Would you like to use TypeScript? No / Yes
Which linter would you like to use? ESLint / Biome / None
Would you like to use React Compiler? No / Yes
Would you like to use Tailwind CSS? No / Yes
Would you like your code inside a `src/` directory? No / Yes
Would you like to use App Router? (recommended) No / Yes
Would you like to customize the import alias (`@/*` by default)? No / Yes
What import alias would you like configured? @/*

Sau khi hoàn tất các câu hỏi, create-next-app sẽ tạo một thư mục theo tên dự án của bạn và cài đặt các dependencies cần thiết.

Cài đặt thủ công (Manual installation)

Để tạo một ứng dụng Next.js mới theo cách thủ công, hãy cài đặt các package cần thiết:

  • pnpm: pnpm i next@latest react@latest react-dom@latest
  • npm: npm i next@latest react@latest react-dom@latest
  • yarn: yarn add next@latest react@latest react-dom@latest
  • bun: bun add next@latest react@latest react-dom@latest

Good to know: App Router sử dụng các bản React canary được tích hợp sẵn, bao gồm tất cả các thay đổi ổn định của React 19 cũng như các tính năng mới đang được kiểm chứng trong các framework. Pages Router sử dụng phiên bản React mà bạn cài đặt trong package.json.

Sau đó, thêm các script sau vào file package.json:

{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "lint": "eslint",
    "lint:fix": "eslint --fix"
  }
}

Các script này tương ứng với các giai đoạn khác nhau trong quá trình phát triển ứng dụng:

  • next dev: Khởi động development server sử dụng Turbopack (bundler mặc định).
  • next build: Build ứng dụng cho môi trường production.
  • next start: Khởi động production server.
  • eslint: Chạy ESLint.

Turbopack hiện là bundler mặc định. Để sử dụng Webpack, chạy next dev --webpack hoặc next build --webpack. Xem tài liệu Turbopack: https://nextjs.org/docs/app/api-reference/turbopack

Tạo thư mục app

Next.js sử dụng cơ chế file-system routing, nghĩa là các route trong ứng dụng được xác định dựa trên cách bạn tổ chức các file.

Tạo một thư mục app. Sau đó, bên trong app, tạo file layout.tsx. Đây là root layout. File này bắt buộc phải có và phải chứa các thẻ <html> và <body>.

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}

Tạo trang chủ app/page.tsx với một số nội dung ban đầu:

export default function Page() {
  return <h1>Hello, Next.js!</h1>
}

Cả layout.tsx và page.tsx sẽ được render khi người dùng truy cập vào đường dẫn gốc của ứng dụng (/).

Good to know:

  • Nếu bạn quên tạo root layout, Next.js sẽ tự động tạo file này khi chạy development server bằng next dev.
  • Bạn có thể tùy chọn sử dụng thư mục src ở thư mục gốc của dự án để tách mã nguồn ứng dụng khỏi các file cấu hình.

Tạo thư mục public (optional)

Tạo một thư mục public ở thư mục gốc của dự án để lưu trữ các tài nguyên tĩnh như hình ảnh, font chữ, v.v. Các file bên trong public có thể được tham chiếu trong mã nguồn của bạn bắt đầu từ đường dẫn gốc (/).

Bạn có thể tham chiếu các tài nguyên này bằng đường dẫn gốc (/). Ví dụ: public/profile.png có thể được tham chiếu là /profile.png:

import Image from 'next/image'

export default function Page() {
  return <Image src='/profile.png' alt="Profile" width={100} height={100} />
}

Chạy development server

  • Chạy npm run dev để khởi động development server.
  • Truy cập http://localhost:3000 để xem ứng dụng của bạn.
  • Chỉnh sửa file app/page.tsx và lưu lại để xem kết quả được cập nhật trong trình duyệt.

Thiết lập TypeScript

Phiên bản TypeScript tối thiểu: v5.1.0

Next.js có hỗ trợ TypeScript tích hợp sẵn.

Để thêm TypeScript vào dự án, hãy đổi tên một file thành .ts hoặc .tsx và chạy next dev. Next.js sẽ tự động cài đặt các dependency cần thiết và thêm file tsconfig.json với các tùy chọn cấu hình được khuyến nghị.

IDE Plugin

Next.js bao gồm một plugin TypeScript tuỳ chỉnh và bộ kiểm tra kiểu (type checker), mà VSCode và các trình soạn thảo mã khác có thể sử dụng để kiểm tra kiểu nâng cao và tự động hoàn thành (auto-completion).

Bạn có thể bật plugin trong VS Code bằng cách:

  1. Mở Command Palette (Ctrl/⌘ + Shift + P)
  2. Tìm kiếm "TypeScript: Select TypeScript Version"
  3. Chọn "Use Workspace Version"

Xem trang tài liệu tham khảo về TypeScript để biết thêm thông tin: https://nextjs.org/docs/app/api-reference/config/typescript

Thiết lập linting

Next.js hỗ trợ linting với ESLint hoặc Biome. Chọn một công cụ lint và chạy trực tiếp thông qua các script trong package.json.

Sử dụng ESLint (bộ quy tắc đầy đủ):

{
  "scripts": {
    "lint": "eslint",
    "lint:fix": "eslint --fix"
  }
}

Hoặc sử dụng Biome (linter + formatter tốc độ cao):

{
  "scripts": {
    "lint": "biome check",
    "format": "biome format --write"
  }
}

Nếu dự án của bạn trước đây sử dụng next lint, hãy chuyển các script sang ESLint CLI bằng codemod sau:

npx @next/codemod@canary next-lint-to-eslint-cli .

Nếu bạn sử dụng ESLint, hãy tạo một file cấu hình rõ ràng (khuyến nghị dùng eslint.config.mjs). ESLint hỗ trợ cả định dạng cũ .eslintrc.* và định dạng mới eslint.config.mjs.

Xem tài liệu tham khảo ESLint API để biết cấu hình được khuyến nghị: https://nextjs.org/docs/app/api-reference/config/eslint#with-core-web-vitals

Lưu ý: Bắt đầu từ Next.js 16, lệnh next build không còn tự động chạy linter nữa. Thay vào đó, bạn có thể chạy linter thông qua các script NPM.

Xem trang tài liệu về ESLint Plugin để biết thêm thông tin: https://nextjs.org/docs/app/api-reference/config/eslint

Thiết lập Absolute Imports và Module Path Aliases

Next.js hỗ trợ sẵn các tùy chọn "paths" và "baseUrl" trong file tsconfig.json và jsconfig.json. Các tùy chọn này cho phép bạn tạo alias cho thư mục dự án thành đường dẫn tuyệt đối, giúp việc import module dễ dàng và gọn gàng hơn. Ví dụ:

// Before

import { Button } from '../../../components/button'

// After

import { Button } from '@/components/button'

Cấu hình Absolute Imports

Để cấu hình import tuyệt đối, thêm tùy chọn baseUrl vào file tsconfig.json hoặc jsconfig.json. Ví dụ:

{
  "compilerOptions": {
    "baseUrl": "src/"
  }
}

Ngoài việc cấu hình baseUrl, bạn có thể sử dụng tùy chọn "paths" để tạo "alias" cho đường dẫn module. Ví dụ, cấu hình sau ánh xạ @/components/* tới components/*:

{
  "compilerOptions": {
    "baseUrl": "src/",
    "paths": {
      "@/styles/*": ["styles/*"],
      "@/components/*": ["components/*"]
    }
  }
}

Mỗi đường dẫn trong "paths" đều được tính tương đối so với vị trí baseUrl.
